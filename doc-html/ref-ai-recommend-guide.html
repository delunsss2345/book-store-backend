Huong Dan Phuong Phap 1: AI Recommend tu orderItem + userEvent

Cap nhat: 2026-02-15.

Muc tieu tai lieu:

  • Giai thich ro "phuong phap 1" la gi.
  • Cho ban mot lo trinh trien khai thuc te, khong can LLM trong runtime.
  • Bam sat du lieu hien co cua project (order_items, orders, user_events, books, book_variants).

#1. Phuong phap 1 la gi?

Phuong phap 1 la recommend kieu "AI ranking tu hanh vi", cu the:

  • Hoc tu hanh vi mua that (orderItem): tin hieu manh nhat.
  • Hoc tu hanh vi tuong tac (userEvent): xem, them gio, checkout, thanh toan...
  • Gom cac ung vien sach phu hop, roi xep hang theo diem.

No la AI vi he thong dang hoc mau hanh vi tu du lieu, khong phai hard-code thu cong danh sach sach.

Luu y:

  • Day KHONG phai la LLM recommend theo prompt.
  • Day la recommendation engine theo du lieu giao dich + event (on dinh, re, de kiem soat).

#2. Tai sao nen dung cho he thong cua ban?

Ban da co san:

  • OrderItem + Order status de biet "mua that".
  • UserEvent de biet "quan tam".
  • Catalog va cache de phuc vu API nhanh.

Nen co the di theo huong:

  • Batch tinh diem dinh ky.
  • API lay ket qua tu bang da tinh san.
  • Cache de latency thap.

#3. Dau vao du lieu (hien co trong schema)

Nguon 1 - Purchase signal (manh):

  • Tu order_items join orders.
  • Chi lay don hop le: PAID, CONFIRMED, PACKING, SHIPPING, DELIVERED.
  • Muc dich: biet sach nao duoc mua cung nhau, sach nao co ty le chuyen doi cao.

Nguon 2 - Behavior signal (nhanh, phong phu):

  • Tu user_events.
  • Event uu tien:
  • VIEW_BOOK
  • ADD_TO_CART
  • CHECKOUT_START
  • PLACE_ORDER
  • PAYMENT_SUCCESS
  • Event phat: REMOVE_FROM_CART, PAYMENT_FAILED

Nguon 3 - Product context:

  • Category, price band, stock, isActive, language.
  • Dung de loc va bo sung fallback.

#4. Luong recommend chi tiet (theo code hien tai)

#4.1 Input contract cho 2 luong event

  • Luong bookVariant:

  • objectType = "bookVariant", objectId = <bookVariantId>.

  • Event duoc tinh diem: VIEW_BOOK, ADD_TO_CART, REMOVE_FROM_CART.

  • Luong order:

  • objectType = "order", objectId = <orderId>.

  • Event duoc tinh diem: CHECKOUT_START, PLACE_ORDER, PAYMENT_SUCCESS, PAYMENT_FAILED.

#4.2 Sequence online (request-time) cho endpoint recommend

  1. Controller goi UserEventService.getRecommend(userId).
  2. Service build song song 2 map diem:
  • buildScoreWithRecommendVariant(userId) -> map <variantId, score>.
  • buildScoreWithRecommendOrder(userId) -> map <orderId, score>.
  1. Service merge 2 map:
  • mergerRecommendVariantAndOrder(variants, orders).
  • Quy trinh merge:
  • lay orderIds tu map order.
  • query OrderRepository.findOrderItemsByOrderId(orderIds).
  • flatten item theo bookVariantSnapshotId.
  • map snapshotId -> bookVariantId qua BookSnapshotRepository.
  • cong diem order vao map variant (co nhan quantity).
  1. Sau merge, dung variantIds lam seed:
  • doi variantIds -> snapshotIds.
  • doi snapshotIds -> orderIds (cac order co chua seed).
  • group co-buy candidate snapshot ids:
  • OrderItemRepository.groupAlsoBoughtSnapshotCandidates(...).
  1. Materialize candidate:
  • doi candidateSnapshotIds -> candidateVariantIds.
  • dedupe variant ids.
  • query CatalogRepository.findBooksVariantByIds(...).
  1. Tra danh sach recommend.
  2. Neu khong co du lieu o bat ky stage nao, return [] ngay (fail-fast).

#4.3 Tai sao merge 2 luong la "diem hay"

  • Luong bookVariant bat duoc "interest signal" ngay lap tuc (view/add/remove).
  • Luong order bat duoc "purchase-intent signal" manh hon (checkout/payment).
  • Sau merge, tat ca diem quy ve cung key bookVariantId, nen candidate generation nhat quan, de mo rong va de debug.
  • He thong deterministic, explainable, khong phu thuoc LLM runtime.

#4.4 Quy uoc de tranh leak chat luong du lieu

  • Bat buoc thong nhat format objectType va objectId nhu muc 4.1.
  • Khong tron id type trong cung objectType (vd: bookId nhung ghi vao bookVariant).
  • Khong ghi event "order" neu chua co orderId hop le.
  • Uu tien su dung event timestamp goc de score/time-decay dung ngu nghia.

#4.5 Definition of done cho workflow hien tai

  • Co du 2 map diem (variant, order) va merge thanh mot map variant.
  • Candidate co-buy sinh tu seed sau merge, khong chi tu mot luong.
  • Endpoint return ket qua on dinh cho:
  • user co event variant nhieu,
  • user co event order nhieu,
  • user co ca hai loai event,
  • user khong co du lieu (fallback []).

#5. Cong thuc diem de bat dau (ban de van hanh)

Goi y trong so event:

  • VIEW_BOOK = +1
  • ADD_TO_CART = +4
  • CHECKOUT_START = +6
  • PLACE_ORDER = +8
  • PAYMENT_SUCCESS = +10
  • REMOVE_FROM_CART = -3
  • PAYMENT_FAILED = -4

Time decay:

  • Diem su kien giam dan theo thoi gian (vi du nua doi 21 ngay).

Diem tong:

  • Score = 0.50 * OrderCF + 0.30 * Behavior + 0.15 * Trend7d + 0.05 * Freshness

Y nghia:

  • OrderCF: "nguoi mua sach A hay mua them sach nao".
  • Behavior: "user nay dang quan tam gi gan day".
  • Trend7d: tranh qua ca nhan hoa trong du lieu it.
  • Freshness: co hoi cho sach moi.

#6. Cold-start va fallback

Cold-start user (chua co lich su):

  • Dung bestSeller + topRated + newArrivals.
  • Neu co guest session event thi dung guest history tam.

Cold-start item (sach moi):

  • Day item moi vao nhom newArrivals.
  • Boost nhe trong 7-14 ngay dau.

Data qua it:

  • Tang ti le trend/fallback.
  • Giam trong so personalization.

#7. Muc tieu chat luong (de do hieu qua)

Online KPI:

  • CTR o block recommend.
  • Add-to-cart rate tu recommend.
  • Conversion rate tu recommend.
  • GMV/session do recommend dong gop.

Offline KPI:

  • Recall@K (do phu item user co mua).
  • NDCG@K (thu hang co hop ly khong).

Moc danh gia de bat dau:

  • Sau 2 tuan A/B test, neu CTR tang >= 10% va conversion tang >= 3-5% thi dat.

#8. Lo trinh trien khai de khong "vo tran"

Phase 1 (3-5 ngay): Baseline

  • Dung trong so event + bestSeller fallback.
  • Tra recommend cho catalog home.
  • Chua can model ML nang cao.

Phase 2 (1-2 tuan): Hybrid day du

  • Them co-buy matrix tu orderItem.
  • Them batch job tinh san top N theo user.
  • Luu ket qua vao bang recommendation de doc nhanh.

Phase 3 (2-4 tuan): Nang cap

  • Learning-to-rank (LightGBM/XGBoost) voi feature tu order + event + context.
  • Explainability ("vi ban da xem/mua...").

#9. Van hanh va chi phi

Tai sao cach nay "on":

  • Khong goi LLM cho moi request.
  • Chi phi thap, latency thap, ket qua on dinh.
  • De debug va de giai thich.

Tan suat batch goi y:

  • Moi 15-60 phut tuy traffic.
  • Neu traffic thap, 2-4 lan/ngay van du.

Cache:

  • Cache ket qua recommend 1-5 phut cho endpoint home.
  • Co invalidation khi co dot sale lon.

#10. Hieu nhanh trong 1 cau

"Phuong phap 1" = dung AI ranking dua tren du lieu mua hang that (orderItem) + du lieu hanh vi (userEvent), tinh diem va xep hang top N; LLM chi la lop bo sung, khong phai dong co chinh.

#11. Checklist truoc khi bat dau lam

  • Da ghi day du userEvent cho cac hanh vi quan trong.
  • Da thong nhat map tu event -> bookId/bookVariantId.
  • Da co fallback cho user moi.
  • Da co KPI va cach do truoc/sau.
  • Da co lich batch + cache key ro rang.
