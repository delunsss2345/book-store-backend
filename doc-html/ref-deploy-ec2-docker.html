Ke hoach deploy BE NestJS len AWS EC2 (Docker, click-by-click, chi phi re)

Cap nhat theo tai lieu AWS den ngay 2026-02-09.

#1) Muc tieu

  • Deploy backend NestJS len EC2 de chay production on dinh.
  • Uu tien chi phi re va de van hanh (ban dang co AWS credit $99).
  • Co domain rieng + HTTPS.
  • SSH an toan vao server.
  • Khong over-engineer: 1 EC2 + Docker Compose + Nginx.

#2) Dinh huong chi phi (voi credit $99)

Thong diep chinh:

  • Khong ep "free tuyet doi".
  • Muc tieu la goi re + on dinh production.

Chi phi du kien (muc tham khao, can check lai truoc khi launch):

  • EC2 t4g.small + EBS gp3 30GB + Public IPv4 + bang thong.
  • Tong chi phi thuong o muc thap-trung binh/thang cho 1 moi truong nho.

Canh bao chi phi quan trong:

  • Public IPv4 tren AWS co phi theo gio.
  • Neu dung Route 53 thi co phi hosted zone + DNS query.

Khuyen nghi truoc khi launch:

  • Dung AWS Pricing Calculator de estimate theo region us-east-1.

#3) Cau hinh da chot cho du an nay

  • Region: us-east-1.
  • Instance type: t4g.small.
  • AMI/OS: Ubuntu Server 24.04 LTS (64-bit Arm).
  • Storage: gp3 30 GiB.
  • Network inbound:
    • 22/tcp chi mo My IP (x.x.x.x/32).
    • 80/tcp mo Anywhere IPv4.
    • 443/tcp mo Anywhere IPv4.
  • DNS: giu nha cung cap DNS hien tai, tro A record ve Elastic IP cua EC2.
  • Runtime: Docker Compose gom app, mysql, redis.

#4) Luong bam nut AWS Console (click-by-click)

#B1. Chon region

  • O goc phai tren Console AWS, chon: US East (N. Virginia) us-east-1.

#B2. Bat billing alert

  • Billing & Cost Management -> Billing preferences -> Alert preferences -> Edit.
  • Bat Receive AWS Free Tier usage alerts.
  • Bam Update.

#B3. Tao budget 0 USD

  • Billing & Cost Management -> Budgets -> Create budget.
  • Chon Use a template (simplified).
  • Chon Zero spend budget.
  • Nhap email nhan canh bao.
  • Bam Create budget.

#B4. Launch EC2

  • EC2 -> Instances -> Launch instances.

#B5. Dien form "Launch an instance"

  • Name and tags: nest-auth-prod.
  • Application and OS Images (AMI): Ubuntu Server 24.04 LTS (64-bit Arm).
  • Instance type: t4g.small.
  • Key pair (login):
    • Create new key pair.
    • Type: RSA.
    • Format: .pem.
    • Bam Create key pair va luu file key.
  • Network settings -> Edit:
    • VPC: default.
    • Auto-assign public IP: Enable.
    • Firewall: Create security group.
    • Rule 1: SSH, Port 22, Source My IP.
    • Rule 2: HTTP, Port 80, Source Anywhere IPv4.
    • Rule 3: HTTPS, Port 443, Source Anywhere IPv4.
  • Configure storage: gp3, 30 GiB.
  • Summary: Number of instances = 1.
  • Bam Launch instance.

#B6. Gan IP tinh (Elastic IP)

  • EC2 -> Elastic IPs -> Allocate Elastic IP address -> Allocate.
  • Chon IP vua tao -> Actions -> Associate Elastic IP address.
  • Chon instance nest-auth-prod -> Associate.

  • EC2 -> Instances -> chon nest-auth-prod -> Connect -> tab SSH client.

#B8. Tro domain o nha dang ky hien tai

  • Vao trang DNS cua nha dang ky domain ban dang dung.
  • Tao record:
    • Type: A
    • Host: api (hoac subdomain ban muon)
    • Value: <Elastic-IP>

#9) Test cases / acceptance criteria

  • EC2 running, da associate Elastic IP.
  • SSH thanh cong bang .pem, login duoc ubuntu, sau do su dung user deploy.
  • Security group dung 3 rule inbound: 22 (My IP), 80, 443.
  • docker compose ps thay app, mysql, redis dang chay.
  • prisma migrate deploy thanh cong.
  • Nginx reverse proxy hoat dong.
  • Certbot cap SSL thanh cong.
  • API truy cap duoc:
    • GET https://api.yourdomain.com/api/v1
    • GET https://api.yourdomain.com/api/v1/docs
  • Billing alert + zero spend budget da tao, email canh bao da cau hinh.
  • DNS resolve dung ve Elastic IP.

#12) CI/CD GitHub Actions (chi tiet cho nguoi moi, huong production)

Muc tieu:

  • Moi Pull Request deu phai qua CI (lint, test, build).
  • Moi lan merge vao main thi deploy tu dong len EC2.
  • Co co che chan deploy nham (Environment production + concurrency).

Cach doc phan nay:

  • Moi muc deu co 2 phan: lam gi va tai sao.
  • Ban co the copy tung khoi lenh theo thu tu, khong nen bo qua.

#12.1 Tong quan pipeline

Luong de xuat:

  1. Dev push code len nhanh feature.
  2. Tao Pull Request vao main.
  3. Workflow CI chay va phai xanh.
  4. Merge PR vao main.
  5. Workflow CD tu dong deploy len EC2.
  6. Chay migration va health check sau deploy.

Tai sao:

  • Tach CI va CD giup tranh dua code loi len production.
  • Chi deploy tu main de giam rui ro sai nhanh.
  • Health check sau deploy de xac nhan app that su len duoc.

#12.3 Chuan bi tren GitHub (Settings)

#A. Tao Environment production

  1. Vao Repo -> Settings -> Environments -> New environment.
  2. Dat ten: production.
  3. (Khuyen nghi) bat Required reviewers de chan deploy nham.
  4. Save.

Tai sao:

  • Environment cho phep dat gate rieng cho deploy production.
  • Required reviewers them 1 lop duyet an toan truoc khi CD chay.

#B. Tao secrets cho deploy

Vao Repo -> Settings -> Secrets and variables -> Actions -> New repository secret.

Tao cac secret sau:

  • EC2_HOST: Elastic IP cua EC2 (vd 54.xx.xx.xx).
  • EC2_USER: deploy.
  • EC2_SSH_PRIVATE_KEY: toan bo private key PEM dung de SSH tu GitHub Actions.
  • EC2_APP_DIR: /srv/nest-auth.
  • EC2_PORT: 22 (neu dung port khac thi doi gia tri).

Tai sao tung secret:

  • EC2_HOST: de workflow biet ket noi may nao.
  • EC2_USER: user dung de SSH va chay docker compose.
  • EC2_SSH_PRIVATE_KEY: xac thuc an toan, tranh hard-code key trong repo.
  • EC2_APP_DIR: giup script deploy khong bi sai duong dan.
  • EC2_PORT: linh hoat neu ban doi SSH sang port khac de hardening.

Luu y bao mat:

  • Khong paste secret vao file code.
  • Khong log key ra workflow.
  • Rotate key neu nghi ngoi lo.

#C. Bat branch protection cho main

  1. Repo -> Settings -> Branches -> Add rule.
  2. Branch name pattern: main.
  3. Bat:
    • Require a pull request before merging.
    • Require status checks to pass before merging -> chon check CI / ci.
    • Require branches to be up to date before merging.
  4. Save changes.

Tai sao:

  • Branch protection buoc team di qua PR + CI, tranh push thang gay vo production.
  • Rule "up to date" dam bao code merge da qua CI tren state moi nhat cua main.

#12.4 Tao workflow CI (.github/workflows/ci.yml)

Tao file:

Ghi chu:

  • Job CI / ci la check can chon trong branch protection.
  • Neu test hien tai phu thuoc DB/Redis that, bo sung service containers o workflow sau.
  • Dung npx eslint ... thay vi npm run lint vi script local dang co --fix; trong CI production, nen kiem tra khong tu sua file.

Giai thich nhanh file CI:

  • Trigger pull_request de chan loi truoc merge, trigger push main de tiep tuc giu main "xanh".
  • permissions: contents: read la nguyen tac it quyen nhat (least privilege).
  • npm ci tao moi truong dependency tai lap duoc, dung cho CI.
  • --runInBand giup test on dinh hon tren runner nho (doi lai co the cham hon).

#12.5 Tao workflow CD (.github/workflows/cd.yml)

Tao file:

Ghi chu production:

  • concurrency dam bao khong deploy chong len nhau.
  • environment: production cho phep them buoc phe duyet thu cong (neu ban bat rule).
  • workflow_dispatch de deploy manual khi can.

Giai thich nhanh file CD:

  • CD khong build tren local may ban, ma build ngay tren EC2 de giam sai khac moi truong.
  • ssh-keyscan de them host key vao known_hosts, tranh loi xac thuc host lan dau.
  • git pull --ff-only dam bao server dung dung commit tren main.
  • curl .../api/v1/docs la health check don gian sau deploy.

#12.7 Rollback nhanh khi deploy loi

Rollback theo commit truoc:

Khuyen nghi:

  • Neu co migration nguy hiem, can chien luoc rollback DB rieng (backup truoc deploy).

Tai sao:

  • Rollback code khong tu dong rollback du lieu DB.
  • Neu migration da thay doi schema/du lieu, phai co backup de quay lai an toan.

#12.8 Checklist CI/CD production

  • Branch protection da bat cho main.
  • CI bat buoc truoc merge.
  • CD chi deploy khi merge main (hoac manual dispatch).
  • Deploy key/SSH key da luu trong GitHub Secrets.
  • Environment production da tao va (neu can) co required reviewer.
  • Co rollback runbook va backup DB.

#12.9 Loi thuong gap (nguoi moi hay gap) + cach xu ly

  1. Permission denied (publickey) khi CD SSH vao EC2.
  • Kiem tra secret EC2_SSH_PRIVATE_KEY co dung key private khong.
  • Kiem tra public key tuong ung da co trong /home/deploy/.ssh/authorized_keys.
  1. Host key verification failed.
  • Dam bao step ssh-keyscan chay dung EC2_HOST va EC2_PORT.
  1. CD thanh cong nhung API 502/timeout.
  • Vao EC2 check docker compose ... ps va logs app.
  • Kiem tra Nginx config (nginx -t) va firewall/security group.
  1. prisma migrate deploy fail.
  • Kiem tra DATABASE_URL trong .env.prod.
  • Dam bao container mysql da healthy truoc khi migrate.
  1. CI fail do test phu thuoc DB/Redis.
  • Bo sung service containers trong workflow CI, hoac mock dependency de test don vi.
