#Tại sao phải tách?

Request DTO Response DTO
Validation decorators (@IsEmail, @MinLength) Serialization decorators (@Expose, @Exclude)
Nhận data từ client Format data trả về client
Chứa tất cả input fields Loại bỏ sensitive fields
Dùng với ValidationPipe Dùng với ClassSerializerInterceptor

#3. Entities & Models

#Khi nào dùng Entity?

  • Khi cần thêm methods, computed properties
  • Khi Prisma generated types không đủ
  • Khi cần domain logic riêng

#4. Repository Pattern

#Tại sao cần Repository?

  • Abstract database operations
  • Dễ mock trong testing
  • Switch database dễ dàng hơn
  • Centralize query logic
