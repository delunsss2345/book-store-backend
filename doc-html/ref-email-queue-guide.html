NestJS Email Queue Guide (BullMQ + Outbox)

Cap nhat: 2026-02-08.

Muc tieu: huong dan tich hop queue gui email vao du an nay, dung BullMQ theo docs NestJS, va tan dung bang email_outbox de tang do tin cay.

#1. Hien trang trong du an

Trong du an da co outbox:

  • Bang email_outbox trong Prisma (co status, attempts, lastError, scheduledAt, sentAt).
  • EmailOutboxService chi tao record PENDING.
  • VerificationCodeService da goi EmailOutboxService.createOtpRegisterEmail().

Y nghia: ta chi can noi them queue + worker de lay record tu outbox va gui email.

#2. Kien truc de xuat (production)

Luong co ban:

  1. API nhan request (register/forgot password...).
  2. Tao record trong email_outbox (status = PENDING).
  3. Enqueue job send-email vao BullMQ (payload co outboxId).
  4. Worker doc job, load record, gui email qua mailer (Nodemailer/SES/Sendgrid...).
  5. Cap nhat status -> SENT hoac FAILED, luu sentAt, lastError, tang attempts.

Vi sao can queue:

  • Tach tac vu gui email ra khoi request realtime.
  • Co the retry/backoff, va scale worker rieng.
  • Redis luu job, worker co the chay doc lap.

#3. Cai dat BullMQ (theo NestJS docs)

Cai package:

Luu y: NestJS docs ghi Bull dang maintenance, BullMQ la lua chon duoc phat trien tich cuc, nen uu tien BullMQ.

#4. Module queue (goi y theo cau truc du an)

Tao module JobsModule (hoac EmailQueueModule), dang ky queue va processor.

#5. Producer: enqueue job tu outbox

Goi y: dung jobId theo outbox id de tranh enqueue trung lap.

#6. Processor: worker gui email

Trong MailService, ban co the dung Nodemailer (SMTP/SES/Sendmail...).

#7. Noi queue vao flow hien tai

Trong VerificationCodeService.createRegisterVerification(...):

  • Sau khi EmailOutboxService.createOtpRegisterEmail(), goi EmailProducer.enqueueOutboxEmail(outboxId).
  • Luu y: can tra ve outboxId tu EmailOutboxService.

#8. Production: thay cho "file chay queue lien tuc" (Express)

Trong NestJS, khong can tu tao vong lap chay mai. Dung queue + worker:

  • API process: chi enqueue job.
  • Worker process: chay @Processor() de xu ly job.

Co 2 cach pho bien:

  1. Chay chung trong 1 app (nho) -> ok cho dev.
  2. Tach process (production) -> worker doc lap, scale theo queue.

Neu muon tach worker, co the:

  • Tao entrypoint rieng (vd main.worker.ts) dung NestFactory.createApplicationContext() va chi import JobsModule.
  • Hoac dung manualRegistration + BullRegistrar de bat processor theo dieu kien (ENV WORKER=true).

#9. Cau hinh can thiet

Toi thieu:

  • REDIS_HOST, REDIS_PORT
  • MAIL_HOST, MAIL_PORT, MAIL_USER, MAIL_PASS (neu dung SMTP)

#10. Goi y van hanh

  • Retry/backoff cho email loai PENDING/FAILED.
  • Idempotency: dung jobId theo outbox id.
  • Monitoring: log job fail + alert.
  • Test local: Nodemailer + Ethereal account.
