What this PR does

  • Adds a Redis read-through cache to PermissionsGuard so the warm path no longer hits MariaDB twice per guarded request — role IDs are still read, but each role's permission list is served from cache.
  • Introduces PERMISSION_CACHE_TTL in enum-ttl.constant.ts and back-fills the cache on a miss inside the existing Promise.all over the caller's roles.
  • Keeps the public contract identical: the guard still throws ForbiddenException when the required @RequirePermissions(...) code is absent.
