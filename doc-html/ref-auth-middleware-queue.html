#1. Mapper nhanh: Express -> NestJS

Express (quen thuoc) NestJS tuong ung theo docs Khi nao dung
app.use(authMiddleware) app.use(...) hoac consumer.apply(...).forRoutes(...) Xu ly request/response som, cross-cutting, khong can biet handler
authRequired middleware Guard (CanActivate) Bao ve route theo context cu the (handler/class metadata)
rolesMiddleware Roles decorator + RolesGuard Kiem tra role theo route/controller
chain next() Request lifecycle: middleware -> guards -> interceptors -> pipes -> handler Hieu dung thu tu de dat logic dung lop

Theo docs NestJS, guard chay sau middleware va truoc interceptor/pipes. Vi vay authz (roles, permissions) nen dat o guard de truy cap duoc ExecutionContext.

Thu tu request lifecycle (rut gon theo docs):

  1. Incoming request.
  2. Middleware (global, module-bound).
  3. Guards (global -> controller -> route).
  4. Interceptors truoc handler (global -> controller -> route).
  5. Pipes (global -> controller -> route -> parameter).
  6. Handler.
  7. Interceptors sau handler (route -> controller -> global).
  8. Exception filters neu co loi.
  9. Server response.

#5. Blueprint de map 1-1 tu Express sang Nest

Neu ban dang co:

  • authMiddleware + authRequired + rolesMiddleware
  • mot worker queue rieng

Thi map sang Nest nhu sau:

  1. middleware: giu lai cac tac vu khong route-aware (request-id, logging, parse thong tin chung).
  2. AuthGuard global (APP_GUARD) + @Public() cho route mo.
  3. RolesGuard + @Roles(...) cho authorization.
  4. request.user duoc set trong auth guard (hoac Passport strategy).
  5. Queue: Producer o service layer, Consumer o @Processor, listener o class rieng.
  6. Neu job CPU/blocking nang: tach sang separate process (processors: [...]).

#6. Checklist production (theo huong dan docs)

  • Secret JWT khong hard-code trong source; lay tu env/config.
  • Password phai hash (bcrypt hoac tuong duong), khong luu plain text.
  • Auth guard global + route public explicit (@Public()).
  • Roles check qua metadata (@Roles) + Reflector.getAllAndOverride.
  • Queue jobs co retry (attempts, backoff) va cleanup (removeOnComplete, removeOnFail).
  • Queue listener/consumer phai dang ky provider day du.
  • Neu dung forked processors, khong phu thuoc DI trong file processor.
