02

Data flow

The PAID transaction stays the source of truth. The outbox insert joins that transaction; the BullMQ enqueue happens only after commit, so a rolled-back payment never produces an email.

Solid = order pipeline. Dashed clay = email delivery. The webhook returns ok as soon as the queue accepts the job — sending never blocks the bank's request.

03

Key code

The two pieces most likely to be done wrong: enqueuing only after the transaction commits, and keeping the job idempotent so a duplicate webhook can't double-send.

src/modules/hooks/hooks.service.ts
// after the PAID $transaction resolves
const { paidOrder } = await this.handleSepayWebhookMarkDone(
  order.id, inbox.id, attempts, providerEventId, body,
);

// resolve recipient: guest email or user email
const to = order.guestEmail ?? order.user?.email;
if (to) {
  const outbox = await this.emailOutbox.createOutboxOrderEmail({
    orderId: paidOrder.id,
    orderCode: paidOrder.orderCode,
    orderStatus: paidOrder.status,
    toEmail: to,
  });
  await this.emailProducer.enqueueOrderEmail(outbox.id);
}
src/modules/jobs/producers/email.producer.ts
async enqueueOrderEmail(outboxId: bigint) {
  await this.emailQueue.add(
    'send-order-email',
    { outboxId },
    {
      attempts: 3,
      backoff: { type: 'exponential', delay: 3000 },
      removeOnComplete: true,
      jobId: `order-email-${outboxId}`, // idempotent
    },
  );
}

Risks & mitigations

Risk
Sev
Mitigation
Duplicate webhook re-enqueues and the customer gets two emails.
HIGH
Deterministic jobId = order-email-{orderId} plus the existing webhook_inbox DONE short-circuit; a re-delivery never reaches the enqueue.
Email enqueued but the PAID transaction later rolls back.
HIGH
Insert the outbox row inside the transaction; call enqueueOrderEmail only after it commits, exactly as the COD branch does.
email_outbox grows unbounded — rows are marked SENT/FAILED but never deleted.
MED
Daily cron prunes terminal rows older than 30 days, following the order.cleanup.job.ts pattern.
SMTP outage parks jobs; customer waits.
LOW
BullMQ already retries 3× with exponential backoff; surface a metric on exhausted jobs for manual resend.
