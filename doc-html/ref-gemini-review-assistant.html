Huong dan chi tiet: AI ho tro viet review nhanh bang Gemini (cho module Review)

Cap nhat: 2026-02-14.

#1. Muc tieu

Ban muon:

  • User nhap bookId, bookVariantId va y chinh (vi du: "san pham te").
  • He thong tu sinh 1 ban nhap review khoang 100 chu.
  • User sua neu can, sau do moi submit review that.

Tai lieu nay huong dan cach tich hop phu hop voi backend NestJS hien tai cua ban.

#2. Luu y quan trong ve "Gemini 4.0"

Tinh den ngay 2026-02-14, tren trang model list Gemini API cong khai khong co model code gemini-4.0/gemini-4.

Model text goi y de bat dau:

  • On dinh, chi phi tot: gemini-2.5-flash
  • Preview manh hon (co the bien dong): gemini-3-flash-preview

Khuyen nghi van hanh:

  • Dung bien moi truong GEMINI_MODEL.
  • Neu sau nay Google mo model "4.0", ban chi doi gia tri env, khong can sua flow.

#3. Kien truc de trien khai trong project nay

Tach 2 API ro rang:

  1. POST /api/v1/reviews/review/draft
  • Chi tao ban nhap review (khong luu DB).
  • Yeu cau dang nhap.
  1. POST /api/v1/reviews
  • API tao review that (ban da co flow check mua hang + chong trung).

Ly do tach:

  • Bao toan nghiep vu (review that chi tao khi user bam submit).
  • De gioi han chi phi AI va de quan sat chat luong ban nhap.

#4. Prerequisites

  1. Tao API key tren Google AI Studio.
  2. Luu key server-side (khong dua ra FE).
  3. Cai SDK:
  • npm install @google/genai
  1. Dat env:
  • GEMINI_API_KEY=...
  • GEMINI_MODEL=gemini-2.5-flash
  • GEMINI_TIMEOUT_MS=8000

#5. Contract de xuat cho API /reviews/review/draft

Request body:

  • bookId (number)
  • bookVariantId (number)
  • userHint (string, vi du: "san pham te")
  • tone (optional, vi du: critical_polite)
  • language (optional, default vi)
  • targetWords (optional, default 100)

Response body:

  • draftText (string)
  • wordCount (number)
  • model (string)
  • safety (object, optional)
  • generatedAt (ISO datetime)

#6. Nguoi dung vao, backend xu ly theo thu tu nao

Pipeline de xu ly 1 request review/draft:

  1. Xac thuc user (lay userId tu access token).
  2. Validate input (bookId, bookVariantId, userHint).
  3. Check quyen review:
  • Check user da mua variant nay trong order_items (qua book_variant_snapshots) hay chua.
  • Neu chua mua: fail ngay (400/403 tuy convention cua ban).
  1. Check trung review:
  • Query 1 lan voi 3 dieu kien: userId + bookId + bookVariantId.
  • Neu da co review: fail ngay (409).
  1. Lay metadata sach de prompt co ngu canh:
  • ten sach, format, nam xuat ban (neu co), category chinh.
  1. Goi Gemini tao ban nhap co cau truc JSON (structured output).
  2. Parse JSON + validate server-side.
  3. Kiem tra so chu:
  • Neu khong dat 100 chu (hoac dung range ban set), goi 1 lan "rewrite to exact 100 words".
  1. Tra ket qua cho FE de user sua va submit API tao review that.

#7. Prompt design cho case "san pham te"

#7.1 He thong (system instruction)

Dat rule ro:

  • Viet bang tieng Viet tu nhien, lich su.
  • Giong dieu tieu cuc nhung khong xuc pham.
  • Khong bia thong tin khong co trong input.
  • Dung khoang 100 chu.
  • Khong chen markdown, emoji, hashtag.

#7.2 User prompt

Truyen:

  • Book metadata (title, format).
  • Y user (userHint = "san pham te").
  • Muc tieu output (100 chu).

Vi du y:

  • "Noi dung bo cuc roi"
  • "Chat luong in kem"
  • "Gia chua tuong xung"

#8. Structured output (rat nen dung)

Y tuong schema JSON:

  • draftText: string
  • wordCount: number
  • sentiment: enum(negative, neutral, positive)
  • needsHumanEdit: boolean

Loi ich:

  • Parse on dinh, it vo format.
  • De validate truoc khi tra ve FE.

#9. Kiem soat "100 chu" cho chat

Ban nen lam 2 lop:

  1. Lop model:
  • Prompt yeu cau "exactly 100 words".
  1. Lop server:
  • Tu dem lai so chu bang bo tach token don gian (split theo whitespace sau khi normalize).
  • Neu lech:
  • Retry 1 lan voi prompt "rewrite to exactly 100 words, keep same meaning".
  • Neu van lech: chap nhan range 95-105 hoac tra loi yeu cau user regenerate (tuy policy).

#10. Safety settings va moderation

Nen bat safetySettings trong request:

  • Hate speech
  • Harassment
  • Sexually explicit
  • Dangerous

Neu model bi block:

  • Tra thong bao than thien: "Noi dung ban nhap chua phu hop, vui long thu lai voi cau tu lich su hon."
  • Khong tra raw blocked content.

#11. Rate limit + cost control

Moc de bat dau:

  • 10 AI draft / user / ngay
  • Timeout 6-8s
  • Retry toi da 1 lan khi timeout
  • Cache nhe theo key (userId + bookVariantId + normalizedHint) trong 3-5 phut

Neu tai cao:

  • Chuyen model ve gemini-2.5-flash.
  • Giam max output tokens.

#12. Logging/observability can co

Moi request can log:

  • userId, bookId, bookVariantId
  • model
  • latencyMs
  • tokenUsage (neu SDK tra ve)
  • blockedReason (neu co)
  • resultWordCount

Khong log full noi dung review neu ban quan tam privacy.

#13. Loi thuong gap va cach xu ly

  1. MODEL_NOT_FOUND
  • Nguyen nhan: dung sai model code (vi du gemini-4.0 chua co public).
  • Cach xu ly: doi sang model code co trong docs.
  1. 400 invalid schema
  • Nguyen nhan: responseJsonSchema sai.
  • Cach xu ly: test schema doc lap truoc.
  1. 429 quota exceeded
  • Nguyen nhan: vuot han muc API key.
  • Cach xu ly: them rate limit noi bo + backoff.
  1. safety blocked
  • Nguyen nhan: prompt/output vi pham nguong safety.
  • Cach xu ly: giam do gay gat trong prompt, giu giong lich su.

#14. Checklist trien khai (khong code)

  1. Tao endpoint POST /reviews/review/draft.
  2. Reuse nghiep vu check "da mua" + "chua review trung".
  3. Tao prompt template co bien (userHint, book metadata, targetWords).
  4. Goi Gemini voi structured output + safety settings.
  5. Validate output + word count.
  6. Tra draftText de FE cho user sua.
  7. User bam gui -> goi POST /reviews de luu review that.
  8. Them rate limit + log + canh bao loi.

#15. Test scenario de ban thu cong

  1. User chua mua variant:
  • Goi /reviews/review/draft -> fail dung rule.
  1. User da mua nhung da review roi:
  • Goi /reviews/review/draft -> fail "khong duoc cmt nua".
  1. User da mua, chua review:
  • Nhap userHint = "san pham te" -> tra ve 1 ban nhap ~100 chu.
  1. Prompt xau/co tu cong kich:
  • He thong block hoac rewrite an toan.
  1. Tai cao:
  • Verify timeout/retry/rate-limit hoat dong dung.
