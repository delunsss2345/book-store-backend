NestJS Cache Guide (Role table)

Cap nhat: 2026-02-09.

Muc tieu: huong dan cache bang Role theo NestJS docs, khong can code tu A-Z.

#1. Khi nao cache Role

  • Bang Role it thay doi, doc nhieu. Cache la hop ly.
  • Can co co che invalidate khi Role bi thay doi.

#2. Cai dat theo docs NestJS

  • Cai package: @nestjs/cache-manager va cache-manager.
  • Bat in-memory cache bang CacheModule.register().
  • TTL mac dinh = 0 (khong het han). Co the set TTL theo milliseconds.

#3. Bat CacheModule (dung toan app)

#4. Cache trong service (goi y, khong full code)

  • Key goi y: roles:all, roles:name:<name>.
  • Flow goi y:
  1. Doc cache -> neu co thi return.
  2. Neu khong co -> query DB.
  3. Set cache -> return.

Inject CACHE_MANAGER de thao tac get/set/del tren cache store.

#5. Auto-cache cho GET endpoint (neu dung controller)

  • Dung CacheInterceptor o controller hoac global.
  • Chi cache GET. Route dung @Res() khong dung duoc CacheInterceptor.
  • Co the dat @CacheKey() va @CacheTTL() de override key va TTL.

#6. Redis cache (theo docs NestJS)

  • Cai them @keyv/redis.
  • Dang ky CacheModule voi stores.
  • Store dau tien la default, cac store sau la fallback.
  • Neu chi muon Redis, co the de lai 1 store Redis trong stores. (goi y)

#7. Invalidate cache khi Role thay doi

  • Khi create/update/delete Role: xoa roles:all va roles:name:<name>.
  • Co the xoa toan bo cache (clear) neu khong quan tam chi tiet.
